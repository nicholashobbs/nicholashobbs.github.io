Parity - in a binary word: 1 if the number of 1s is odd, otherwise it is 0. Often used for detecting single bit errors in data storage, communication. A parity bit is an appended bit which is 1 if the number of 1s is odd in even parity (even number of total 1s) or 1 if the number of 1s is even in odd parity (odd number of total 1s).

Bit-shift tricks - $x \& (x-1)$ is x with lowest set bit (1) erased. $x \& \sim (x-1)$ isolates the lowest 1 in x.

  1. Parity of a word
  2. How would you compute parity for a large number of 64 bit binary words?
  3. Test each new bit while tracking the number of 1s seen so far - check whether result to the power of (current bit & 1) returns 0 or 1, shift binary to the right and check for each.
  4. define, set result 0, while loop makes result = result to the power of 1 for odd, 0 for even, right bit shift, repeat for each bit, return result
  5. def parity(x): result = 0 while x: result ^= x & 1 x >>= 1 return result

Erase the lowest set bit to reduce time complexity to O(k): def parity(x): result = 0 while x: result ^= 1 x &= x - 1 return result Even better, use caching to check the parity of every 16 bit subword and then the overall parity. This allows for all the results to be cached. Use masks in such a way that you right shift by a certain amount each time, checking and storing parity along the way. def parity(x): MASK_SIZE = 0 BIT_MASK = 0xFFFF return (PRECOMPUTED_PARITY[x >> (3 * MASK_SIZE)] ^ PRECOMPUTED_PARITY[(x >> (2 * MASK_SIZE)) & BIT_MASK] ^ PRECOMPUTED_PARITY[(x >> MASK_SIZE) & BIT_MASK] ^ PRECOMPUTED_PARITY[x & BIT_MASK]) The time complexity is O(n/L) where n is the word size, L is the width of words for which results are cached. XOR can be used since it is 0 if both bits are 0 or 1, and 1 otherwise. XOR for a group of bits is its parity.
